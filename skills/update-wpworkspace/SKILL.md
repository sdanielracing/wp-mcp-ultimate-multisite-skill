# Skill: update-wpworkspace

Updates an existing workspace. Handles two scenarios: updating only the business profile (when credentials are still valid), or doing a full reset including new credentials (when the Application Password was revoked or regenerated).

## Invocation

```
/update-wpworkspace [name]
/update-wpworkspace            ← no argument: lists workspaces and asks which to update
```

---

## Step 1 — Find the workspace

Read `~/.config/wp-workspaces/workspaces.json`. If no name was provided, list all workspaces and ask which one to update.

Show a summary of the current workspace so the user knows what they're editing:

```
Updating workspace: [id]

Current config:
  Site:    [site_url]
  Client:  [client.business_name] · [client.industry]
  Contact: [client.phones] · [client.email]
```

---

## Step 2 — Check credential status

```
Has the WordPress Application Password for this site been revoked
or regenerated since this workspace was created?

  A) No — my credentials are still valid
  B) Yes — I generated a new Application Password (or need to reset)
```

---

## Path A — Credentials still valid (business info update only)

Skip straight to updating the business profile. Show what's currently stored:

```
Your current business profile:

  Description: [client.description]
  Tone:        [editorial.tone]
  Audience:    [editorial.audience]
  Avoid:       [editorial.avoid]
  Notes:       [editorial.notes]

What would you like to update?
You can rewrite the full description, update specific contact details,
change the tone, add legal notices — anything. Just tell me what changed.
```

Take the user's input, regenerate the affected fields, show a diff-style summary, and ask to confirm before writing.

---

## Path B — Credentials revoked (full reset)

Run the full credential flow from `/add-wpworkspace` Steps 2–5, then update the business profile:

### Step B1 — New WP MCP configuration

```
Paste the updated MCP server configuration from your .mcp.json:

  "site_url": "https://yoursite.com",
  "wp": {
    "mcp_endpoint": "https://yoursite.com/wp-json/mcp/wp-mcp-ultimate",
    "auth_header": "Basic YOUR_BASE64_HERE",
    "mcp_server": "wp-mcp-ultimate"
  }
```

Extract `site_url` and `mcp_endpoint`. Ignore `auth_header` — it will be recalculated.

### Step B2 — WordPress username

```
What is your WordPress username?
```

### Step B3 — New Application Password

```
What is the new Application Password generated by WordPress?
(WP Admin → Users → Your Profile → Application Passwords → Generate New)
```

### Step B4 — Calculate and confirm Base64

Calculate:
```bash
echo -n "username:application_password" | base64
```

Show result and ask for confirmation:

```
Your new authorization has been calculated:

  Username:            [username]
  Application Password:[app_password]
  Base64 result:       [calculated_base64]
  auth_header will be: Basic [calculated_base64]

This was auto-calculated to make sure it works correctly.
Confirm this is right before continuing.

✅ Correct — continue
✏️  Re-enter credentials
```

### Step B5 — Business profile update

```
Do you also want to update the business description and editorial profile?
(or keep the current one as-is)
```

If yes, show current profile and follow Path A's update flow.

---

## Step 3 — Write and confirm

Show a summary of all changes (credential update and/or profile changes), ask to confirm, then write to `~/.config/wp-workspaces/workspaces.json`.

```
✅ Workspace "[id]" updated.

Activate now? → /use-wpworkspace [id]
```

---

## Notes

- Only fields that changed are updated — everything else is preserved
- If the site URL changed (e.g. domain migration), treat it as a full reset (Path B)
- To add a brand new workspace: `/add-wpworkspace`
