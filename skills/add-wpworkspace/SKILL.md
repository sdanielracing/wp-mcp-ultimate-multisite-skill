# Skill: add-wpworkspace

Adds a new client workspace through a guided setup. Handles WP credentials securely, auto-calculates the Base64 authorization, and generates the full editorial profile from a plain-text business description.

## Invocation

```
/add-wpworkspace
```

---

## Setup flow — step by step

### Step 1 — Workspace name

```
Let's set up a new workspace.

What would you like to call this workspace?
Use a short slug — lowercase, no spaces (e.g. corner-brew, harmon-legal, janes-studio).
This is the name you'll type in /use-wpworkspace to activate it.
```

Validate: must be lowercase, no spaces or special characters. If the name already exists in workspaces.json, warn and ask for a different one.

---

### Step 2 — WP MCP Ultimate configuration

```
Now paste the MCP server configuration that WP MCP Ultimate generated for your site.
You can find it in your .mcp.json file. It looks like this:

  "site_url": "https://yoursite.com",
  "wp": {
    "mcp_endpoint": "https://yoursite.com/wp-json/mcp/wp-mcp-ultimate",
    "auth_header": "Basic YOUR_BASE64_HERE",
    "mcp_server": "wp-mcp-ultimate"
  }

Paste yours below:
```

Extract `site_url` and `mcp_endpoint` from what the user pastes. Ignore `auth_header` from this paste — it will be recalculated in the next steps.

---

### Step 3 — WordPress username

```
What is your WordPress username?
(The one you use to log into the WP admin dashboard)
```

---

### Step 4 — Application Password

```
What is the Application Password generated by WordPress?
(Go to WP Admin → Users → Your Profile → Application Passwords → Generate New)

This is not your regular login password — it's a separate API key that looks like:
  xxxx xxxx xxxx xxxx xxxx xxxx
```

---

### Step 5 — Calculate and confirm Base64 authorization

Using the username from Step 3 and the Application Password from Step 4, calculate:

```
base64("username:application_password")
```

Use the Bash tool to compute it:
```bash
echo -n "username:application_password" | base64
```

Then show the result to the user and ask for confirmation:

```
Your authorization has been calculated:

  Username:            [username]
  Application Password:[app_password]
  Base64 result:       [calculated_base64]

  auth_header will be: Basic [calculated_base64]

This was auto-calculated from your credentials to make sure it works correctly.
Please confirm this is the right account before we continue.

✅ Looks good — continue
✏️  Something is wrong — let me re-enter my credentials
```

If the user wants to re-enter, go back to Step 3.

---

### Step 6 — Business description

```
Almost done. Tell me about this client.

Write as much or as little as you like — the more detail you give, the better
Claude will perform without needing extra instructions later. Include:

  • What the business does and what makes it unique
  • Who their customers or audience are
  • The tone and voice they use (formal, friendly, authoritative, playful...)
  • What language and region they operate in
  • Any content rules — things to always say, always avoid, required disclaimers
  • Contact hours, service areas, specialties, certifications, or anything else
    that should appear in content, CTAs, footers, or structured data (schema markup)

This description is used to automatically generate the editorial profile and
JSON-LD schema for the site — contact info, business type, and content rules
will be embedded consistently across every post and page without repeating them.
```

---

### Step 7 — Generate and confirm workspace

From the description, generate:
- `client.industry` — inferred sector
- `client.description` — clean 1–2 sentence summary
- `editorial.tone`, `voice`, `language`, `audience`, `avoid`, `cta_style`, `notes`

Show the full workspace as a readable summary and ask:

```
Here's the workspace I've built:

[formatted summary — not raw JSON]

Save this workspace? (yes / adjust first)
```

On confirmation: read `~/.config/wp-workspaces/workspaces.json`, append the entry, write back.

---

### Step 8 — Confirm and activate

```
✅ Workspace "[id]" saved.

Activate now? → /use-wpworkspace [id]
```

---

## Generated entry structure

```json
"[workspace-id]": {
  "site_url": "[https://url.com]",
  "wp": {
    "mcp_endpoint": "[https://url.com/wp-json/mcp/wp-mcp-ultimate]",
    "auth_header": "Basic [calculated_base64]",
    "mcp_server": "wp-mcp-ultimate"
  },
  "client": {
    "business_name": "[from description]",
    "contact_name": "[from description]",
    "industry": "[inferred]",
    "description": "[summary]",
    "phones": ["[from description]"],
    "whatsapp": "[from description or empty string]",
    "email": "[from description or empty string]",
    "address": "[from description or empty string]",
    "site_web": "[site_url]",
    "legal": ["[any notices from description, or empty array]"]
  },
  "editorial": {
    "tone": "[generated]",
    "voice": "[generated]",
    "language": "[generated]",
    "audience": "[generated]",
    "avoid": ["[generated]"],
    "cta_style": "[generated]",
    "notes": "[generated]"
  }
}
```

---

## Notes

- `auth_header` is always calculated in Step 5 — never taken from a pasted config
- Empty optional fields use `""` or `[]`, never omitted
- To update a workspace later: `/update-wpworkspace`
